# Facilidades e Dificuldades

## Introdução

Este documento tem como objetivo mostrar as principais **facilidades** e **dificuldades** que cada integrante do grupo enfrentou durante a disciplina de **Requisitos de Software**.  

Além de servir como um **feedback construtivo** para o professor, ele também busca ajudar **futuros alunos** a entender melhor os **desafios, aprendizados e experiências práticas** que fazem parte da matéria.  

Com isso, esperamos contribuir para uma **melhor organização**, **comunicação** e **compreensão** das etapas do projeto nas próximas turmas.
